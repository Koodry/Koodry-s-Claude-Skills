# Целевая структура и конвенции лор-базы

Эталонная форма проекта, к которой куратор приводит файлы. Это не жёсткий закон, а
карта: если в конкретном проекте уже сложилась похожая, но иначе названная
структура — следуй фактической, а не ломай её под этот образец.

## Содержание
1. Таксономия папок
2. Роли файлов
3. Триггеры выноса (что куда)
4. Конвенции (маркеры, ссылки, индексы)
5. Memory-файлы и ссылки на них
6. Шаблоны файлов

---

## 1. Таксономия папок

```
/  (корень)
  00_Индекс.md              мастер-индекс проекта (точка входа во всё)
  <Справочник системы>.md   правила механики/сеттинга (напр. ChaosGacha)

  Сюжет/
    00_Индекс.md
    Мастер-план_*.md         поэпизодные планы по аркам
    Разбор_Арка*.md          разборы/анализ арок
    Баланс_*.md              заметки по темпу/балансу
    Правки_*.md              рабочие правки глав

  Мир/
    00_Индекс.md
    Мир_космология_и_локации.md
    Силы.md                  таксономия источников силы мира
    Правила_мира.md          канон-континуитет, ограничители, чего НЕТ
    События_и_таймлайн.md
    Организации/             когда конкретная орг разрастается — свой файл
      00_Индекс.md
      <Организация>.md       напр. ЩИТ.md, Озкорп.md, Культ-Даркхолдеров.md
    Локации/                 (по необходимости, та же логика, что Организации)
      00_Индекс.md
      <Локация>.md

  Персонажи/
    00_Индекс.md
    <Имя-Фамилия>.md         карточка персонажа
    Отношения/
      00_Индекс.md
      <A>-×-<B>.md           линия отношений пары

  Способности_ГГ/
    00_Индекс.md             роль индекса может играть журнал
    Способности_ГГ_журнал.md хронологический реестр всех сил ГГ (краткие строки)
    Досье_<Способность>.md   развёрнутое досье одной силы
```

Подпапки уровня сущности (`Мир/Организации/`, `Мир/Локации/`) **создаём только
когда есть что в них класть** — то есть когда хотя бы одна организация/локация
переросла общий файл. Пустых папок не плодим.

## 2. Роли файлов

- **Мастер-индекс (`/00_Индекс.md`)** — единственная точка входа. Перечисляет
  крупные разделы (Мир, Персонажи, Способности, Сюжет, справочник) со ссылками.
- **Индекс папки (`00_Индекс.md`)** — таблица файлов папки: что внутри + статус.
  Плюс краткое правило ведения и ссылки на смежные разделы вне папки.
- **Карточка персонажа** — досье одного героя (данные, силы, характер, связи,
  развитие). Подробные линии отношений выносятся в `Отношения/`.
- **Файл мира** — один аспект мира (космология, силы, правила, таймлайн). Крупные
  отдельные сущности (орг, локация) выносятся в подпапку.
- **Журнал способностей** — краткий хронологический реестр сил ГГ (таблица + сжатые
  карточки). Разросшиеся силы выносятся в `Досье_*`.
- **Досье способности** — полное описание одной силы (механика, расчёты, развитие).
- **Сюжетные файлы** — планы, разборы, правки; живут в `Сюжет/`.

## 3. Триггеры выноса (что куда)

| Что разрослось | Признак | Куда вынести | Что оставить в исходнике |
|---|---|---|---|
| Раздел «Связи»/одна пара в карточке персонажа | подробное описание пары, >~25 строк или >~40% файла | `Персонажи/Отношения/<A>-×-<B>.md` | 2–4 строки сводки + `Подробнее → [[<A>-×-<B>]]` |
| Описание одной силы ГГ в журнале | карточка силы переросла строку реестра | `Способности_ГГ/Досье_<Сила>.md` | строка в таблице + краткая карточка + ссылка на досье |
| Организация в общем файле `Организации.md` | у орг появился развёрнутый блок | `Мир/Организации/<Орг>.md` | абзац-сводка + `Подробнее → [[<Орг>]]` |
| Локация в файле космологии/локаций | то же | `Мир/Локации/<Локация>.md` | абзац-сводка + ссылка |
| Сюжетный/разборный файл в корне | лежит не в `Сюжет/` | `Сюжет/` | — (перемещаем файл целиком, чиним ссылки) |

Принцип: выносим **сущность**, у которой есть собственное место в таксономии, или
блок, переросший дом. Короткое не трогаем.

## 4. Конвенции

**Статус-маркеры** (в индексах и статус-блоках): 🟡 черновик · 🟢 устойчивый канон ·
🔴 пусто/нужно заполнить.

**Пометки в тексте:** `(?)` — спорная/непринятая идея; принятый канон — без пометки.
`(!)` — блок «сверить с внешним источником» (например, черновой импорт фактов
канона Marvel), который надо вычистить после сверки. При переносе блока эти пометки
**едут вместе с ним** — не теряем и не «повышаем» статус самовольно.

**Ссылки:**
- Внутри базы — вики-ссылки `[[Имя-файла-без-расширения]]` (как в карточках) и/или
  относительные markdown-ссылки `[Текст](Путь.md)` (как в индексах). Сохраняй тот
  стиль, который уже принят в конкретном файле/папке.
- Указатель на вынесенный блок оформляй в духе проекта:
  `**Подробнее → [[Морозов-×-Романова]]**` или `Подробная линия → [[...]]`.

**Правило ведения** (повторяется в индексах): один факт меняется — правим в **одном**
файле, остальные ссылаются.

## 5. Memory-файлы и ссылки на них

Проект может опираться на файлы авто-памяти ассистента: папка `memory/` с индексом
`MEMORY.md`, внутри — по одному факту на файл, имена в kebab-case (`bolezn-lor`,
`abilities-log`, `kanon-taimlain-venom`, `chaosgacha-fanfic`). Ссылаются на них из
карточек как `[[слаг]]`, а из файлов мира — инлайном `memory \`слаг\``.

Для куратора это **внешний слой**, а не битые ссылки:
- `[[слаг]]`, совпадающий с файлом памяти, — рабочая ссылка; не чинить, не
  переписывать на внутрипроектный файл.
- memory-файлы по умолчанию не трогаем; редактирование/удаление — только с явного
  согласия автора (раздел «Работа с memory» в SKILL.md).
- при расхождении memory↔файл проекта — предлагаем автору выбор синхронизации, не
  правим молча.

## 6. Шаблоны файлов

Шаблоны — ориентир под стиль проекта, не прокрустово ложе. Подгоняй разделы под
конкретную сущность.

### Индекс папки (`00_Индекс.md`)

```markdown
# <Раздел> — индекс

<1–2 строки: что в этой папке и чего тут НЕ держим (что лежит отдельно).>

## Файлы

| Файл | Что внутри | Статус |
|---|---|---|
| [<Файл>](<Файл>.md) | <краткое описание> | 🟡 черновик |

Статусы: 🟡 черновик · 🟢 устойчивый канон · 🔴 пусто/нужно заполнить

## Связанные файлы проекта (вне папки)
- `<путь>` — <зачем>

## Правило ведения
Один факт меняется — правим в одном файле, остальные ссылаются. Спорное помечаем `(?)`.
```

### Файл отношений (`Персонажи/Отношения/<A>-×-<B>.md`)

```markdown
# <A> × <B> — линия отношений

> **Пара:** <A> × <B>
> **Тип:** <романтическая / соперничество / наставничество …>
> **Статус:** <как развивается>
> **Карточки:** [[<A>]] · [[<B>]]
> **Смежное:** [[<связанный персонаж/файл>]]

---

## 1. Ядро связи
## 2. Движок
## 3. Преграды
## 4. Арка отношений (по аркам)
```

### Досье способности (`Способности_ГГ/Досье_<Сила>.md`)

```markdown
# Досье способности: <Название>

<1 абзац: что это, когда/как получена, роль.>

> **Что это по сути.** <короткое определение/метафора механики>

---

## 1. Системная справка
- **Категория / Редкость / Источник-вдохновение / Энергомодель**
## 2. Механика и расчёты
## 3. Ограничения и баланс
## 4. Развитие по сюжету
```

### Файл организации/локации (`Мир/Организации/<Орг>.md`)

```markdown
# <Организация>

> **Тип / сторона:** <…>
> **Статус:** 🟡 черновик
> **Смежное:** [[<персонажи и файлы>]]

---

## Роль в мире
## Состав / иерархия
## Связи и пересечения
## Открытые узлы
```

### Карточка персонажа (опорные разделы)

Статус-блок (`> Статус / Роль / Фракция / Первое появление / Тип`), затем:
`Персональные данные` · `Характеристики` · `Способности и силы` · `Характер` ·
`Связи` (подробные пары — выносим в `Отношения/`, тут сводка+ссылка) ·
`Развитие по сюжету` · `Заметки автора`. Черновой внешний материал — в блок с `(!)`.
