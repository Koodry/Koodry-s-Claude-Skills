# Writer Skills

Набор скиллов для работы над художественной книгой. 6 скиллов, каждый — отдельный `.skill` файл.

## Скиллы

**character-card** — ведёт карточки персонажей (досье): данные, способности, характер, развитие по сюжету. Один герой — один файл в `Персонажи/`. Различает два режима правки: реткон (правка сути) и фиксация изменений по сюжету.

**plot-check** — разбор сюжета в целом (синопсис, план, глава): логика, дуги персонажей, структура, соответствие целям автора, читательский интерес, баланс темпа для боевых жанров. Два режима: лёгкий (по умолчанию) и тяжёлый — сверка с канон-базой проекта.

**decision-check** — проверка рациональности конкретных поступков персонажей через матожидание (EV): что было бы идеально логично, что правдоподобно для этого героя, и насколько фактический ход отклоняется. Аффективные реакции не разбирает.

**style-check** — диагностика слога: находит корявые места, ставит диагноз по имени (канцелярит, повтор, созвучие…), различает дефект и приём, судит каждый регистр текста по своим правилам. Никогда не переписывает текст целиком — только пометки и 1–2 варианта-иллюстрации в голосе автора.

**project-curator** — библиотекарь лор-базы: реорганизация файлов («один факт — один файл»), вынос разросшихся блоков, битые [[вики-ссылки]], дубли и конфликты фактов, рассинхрон с memory. Меняет файлы только после подтверждения плана.

**feat-judge** — ⚠️ **только для работ по системе ChaosGacha.** Судья фитов: за что герою положен билет, какой тир (Bronze→Divine) и категория (Ability/Trait/Skill/Item/Familiar). Учитывает текущую силу ГГ, правило собственного вклада, отложенные фиты (Changed Fate). Подсказывает, как доработать сцену под лучший тикет.

## Какую модель использовать

Общий принцип: чем больше текста, тоньше суждение или тяжелее режим — тем старше модель (Sonnet → Opus → Fable).

| Скилл | Диапазон | Когда какую |
|---|---|---|
| character-card | Sonnet–Opus | Sonnet — завести/дополнить карточку по шаблону. Opus — сложный реткон или карточка с активным подтягиванием канона из многих файлов. |
| plot-check | Sonnet–Fable | Sonnet — быстрый прогон одной главы. Opus — арка или план книги (лёгкий режим). Fable — тяжёлый режим (сверка с каноном) и разбор всего романа. |
| decision-check | Sonnet–Fable | Sonnet — одно простое решение. Opus — стандарт: расчёт EV с психологией персонажа. Fable — разбор целой главы/арки на десяток решений со сверкой с канон-файлами. |
| style-check | Opus–Fable | Не ниже Opus: нужны слух на ритм и попадание в голос автора. Fable — важные сцены и куски ~800+ слов, где критично не «загрязнить» варианты. |
| project-curator | Sonnet–Opus | Sonnet — скан, битые ссылки, перемещения. Opus — слияние дублей и конфликты фактов, где нужно судить, что оставить. |
| feat-judge | Sonnet–Fable | Sonnet — одиночный фит с ясным входом. Opus — стандарт: калибровка по прецедентам журнала. Fable — целая глава с отложенными фитами (Changed Fate) и спорными тирами. |

## Как выбрать скилл

| Вопрос | Скилл |
|---|---|
| Кто этот персонаж? Запиши/обнови его | character-card |
| Работает ли история? Не провисает ли? | plot-check |
| Логичен ли этот поступок? | decision-check |
| Как это написано? Что звучит криво? | style-check |
| Навести порядок в файлах проекта | project-curator |
| Какой билет дать за сцену? (только ChaosGacha) | feat-judge |
